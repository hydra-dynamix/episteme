Typed Walk Signatures

This shows what a typed walk through a knowledge graph looks like, and how it becomes an identity-free signature. The colors are canonical node ids (0,1,2...), assigned by first appearance in the walk. Repeating colors = recurrence (a node the walk returns to). Node roles (claim/record/method) are shown as small labels but are not part of the signature — only the recurrence pattern and edge types are.

1. Concrete walk (the actual labels)

Each circle is a real node (K-claim, L-record, etc.). Same label = same node revisited.

2. Canonical walk (labels stripped — this IS the signature input)

Labels gone. Each node is now just its first-occurrence id. The pattern of colors = the recurrence shape. Roles are metadata, faded.

3. Collapsed graph (each unique node once)

The same walk drawn as a graph: 3 nodes, edges showing transitions. This is the actual shape.
