Two-Stage Retrieval: Topology Basin + Typed Projection

The substrate stores only label-free topology (the recurrence pattern — no node ids, no edge types). That is the basin. The typed edges (SUPPORTS/WEAKENS/...) are stored as payload and unpacked on demand to refine the basin into a typed graph. Coarse stage = identity-free + label-free recall. Fine stage = typed projection.

Architecture

storage key = label-free canonical signature (recurrence pattern only) payload = typed edges (SUPPORTS, WEAKENS, MEASURES, ...) [NOT part of the key] retrieval: evidence walk -> coarse match on topology [basin: broad, identity-free] -> unpack typed payload -> fine match on typed prefix [projection: narrow, discriminating]
